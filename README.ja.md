# VR鯖江ツクリテ辞典 (VR Sabae Creator's Dictionary)

日本の鯖江市のクリエイターを紹介するウェブベースのVR体験です。このプロジェクトではA-Frameを使用し、職人の写真とプロフィール情報を両面に表示した回転パネルを円柱状に配置して展示します。

## デモ

- [ライブデモ](https://code4fukui.github.io/vr-scc-creators/)

## 特徴

- **没入型VRギャラリー**: A-Frameを使用してクリエイターのプロフィールを3D空間に表示します。
- **両面パネル**: 各パネルの前面にクリエイターの写真、背面に詳細なテキスト情報を表示します。
- **動的画像生成**: HTML-to-Canvas変換により、構造化されたテキストデータを自動的に画像テクスチャに変換して背面パネルに適用します。
- **ローカルキャッシュ**: 初回実行時にDenoサーバーでテキストベースの画像を生成・保存し、`data/`ディレクトリに格納することで、以降の読み込みを高速化します。
- **円形アニメーション**: パネルを多層の円柱状に配置し、ゆっくりと回転させることでダイナミックなブラウジング体験を提供します。

## 仕組み

アプリケーションは公開されたCSVファイルからクリエイターのデータを取得します。各クリエイターごとに、A-Frameの`a-plane`エンティティを背中合わせにしたペアを作成します。

1. **画像生成 (初回実行時)**: `index.html`内の`maketextimg`フラグを`true`に設定すると、`creators.js`スクリプトがクリエイターのデータを含むHTML要素を動的に生成し、PNG画像に変換してBase64エンコードされたデータをローカルのDenoサーバーに送信します。
2. **キャッシュ**: `make_textimg_server.js`サーバーがデータを受信し、デコードして`data/`ディレクトリに`.png`ファイルとして保存します。
3. **通常動作**: `maketextimg`が`false`の場合は、事前に生成された画像を`data/`ディレクトリから直接読み込み、生成ステップをスキップします。

## セットアップと使用方法

このプロジェクトは2段階のセットアッププロセスが必要です。テキスト画像を一度生成し、その後メインアプリケーションを実行します。

### ステップ1: テキスト画像の生成 (初回セットアップ)

このプロセスではローカルのDenoサーバーを使用してクリエイターのデータを画像に変換し、ローカルに保存します。

1. リポジトリをクローンします。
2. `index.html`で`maketextimg`定数を`true`に設定します:
    ```javascript
    const maketextimg = true; // if true, run with make_textimg_server.js
    ```
3. ターミナルからDenoサーバーを起動します。このサーバーは生成された画像を待ち受け、保存します。
    ```sh
    deno run --allow-net --allow-write make_textimg_server.js
    ```
4. `index.html`をウェブブラウザで開きます。ページが読み込まれると、プロジェクトフォルダ内の新しい`data/`ディレクトリに画像ファイルが生成されます。すべてのパネルが表示されればこのステップは完了です。

### ステップ2: 体験の表示

`data/`ディレクトリに画像がキャッシュされると、Denoサーバーなしで体験を実行できます。

1. Denoサーバーを停止します (Ctrl+C)。
2. `index.html`で`maketextimg`定数を`false`に戻します:
    ```javascript
    const maketextimg = false;
    ```
3. 任意のローカルウェブサーバーでプロジェクトフォルダをホストし、`index.html`をブラウザで開きます。
